# Тема: класс Object и метод equals()

В этой папке:

- `Practice1.java` — пример класса с переопределенным методом `equals()`

## Практика 1

В качестве примера придуман класс `Student` с полями:

- `name`
- `age`

Метод `equals()` переопределен так, чтобы два объекта `Student` считались равными, если у них одинаковые:

- имя;
- возраст.

То есть сравнение идет **по содержимому объекта**, а не по ссылке.

## Каким соглашениям должна следовать реализация equals()

Корректная реализация `equals()` должна соблюдать следующие свойства:

### 1. Рефлексивность

Любой объект должен быть равен сам себе:

```java
x.equals(x) == true
```

### 2. Симметричность

Если `x.equals(y)` равно `true`, то и `y.equals(x)` тоже должно быть `true`.

### 3. Транзитивность

Если `x.equals(y)` и `y.equals(z)` равны `true`, то и `x.equals(z)` тоже должно быть `true`.

### 4. Согласованность

Если поля объектов не изменяются, то повторные вызовы `equals()` должны давать один и тот же результат.

### 5. Сравнение с null

Для любого ненулевого объекта:

```java
x.equals(null) == false
```

## Важное замечание

Если переопределяется `equals()`, то обычно нужно переопределять и `hashCode()`.

Это необходимо для правильной работы объектов в коллекциях, например:

- `HashSet`
- `HashMap`

Если два объекта равны по `equals()`, то их `hashCode()` тоже должен быть одинаковым.

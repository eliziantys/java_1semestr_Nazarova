# Тема: super и this

В этой папке:

- `Practice1.java` — пример всех трех вариантов использования `super`
- `Practice3.java` — переписанный код конструкторов с использованием `this()`

## Практика 1

В `Practice1.java` показаны все три случая использования `super`:

1. вызов конструктора суперкласса:

```java
super(parentA);
```

2. доступ к полю суперкласса:

```java
super.a
```

3. вызов метода суперкласса:

```java
super.method();
```

## Практика 2

Если поле `a` и метод `method()` не определены в классе `B`, а определены только в классе `A`, то код все равно будет работать.

Причина в том, что класс `B` наследуется от `A`, а класс `C` наследуется от `B`. Значит, класс `C` косвенно наследует члены класса `A`.

Поэтому в классе `C`:

```java
int a = super.a;
super.method();
```

будет происходить обращение к унаследованным членам, которые пришли из `A` через `B`.

То есть:

- `super.a` даст доступ к полю `a` из `A`;
- `super.method()` вызовет метод `method()` из `A`, если в `B` он не переопределен.

Важно:

`super` обращается к ближайшему суперклассу, то есть к `B`, но если в `B` нужный член не найден, используется унаследованный член из `A`.

## Практика 3

Код переписан через цепочку конструкторов:

- конструктор без параметров инициализирует `z`;
- конструктор с одним параметром вызывает `this()`;
- конструктор с двумя параметрами вызывает `this(a)`;
- конструктор с тремя параметрами вызывает `this(a, b)`.

Это позволяет не повторять одинаковый код инициализации.

# Тема: final-методы и final-классы

В этой папке:

- `Practice1.java` — пример, показывающий, что `final`-метод нельзя переопределить
- `Practice2.java` — пример, показывающий, что от `final`-класса нельзя наследоваться

## Практика 1

Если метод суперкласса объявлен как `final`, то подкласс не может его переопределить.

В `Practice1.java` запрещенный вариант оставлен в комментарии:

```java
@Override
public void show() {
    ...
}
```

Если раскомментировать этот код, программа не скомпилируется.

## Практика 2

Если класс объявлен как `final`, то создать от него подкласс нельзя.

В `Practice2.java` запрещенный вариант тоже оставлен в комментарии:

```java
class Child extends FinalClass {
}
```

Если раскомментировать этот код, программа не скомпилируется.

## Вывод

- `final` у метода запрещает переопределение;
- `final` у класса запрещает наследование.

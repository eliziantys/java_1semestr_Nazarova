# Тема: спецификаторы доступа

В этой папке приведен пример программы, который показывает работу всех четырех вариантов доступа:

- `private`
- без спецификатора доступа
- `protected`
- `public`

## Структура

- `same/AccessDemo.java` — класс с полями и методами всех уровней доступа
- `same/SamePackageTest.java` — пример доступа из класса того же пакета
- `other/ChildInOtherPackage.java` — пример доступа из подкласса в другом пакете
- `other/OtherPackageTest.java` — пример доступа из обычного класса в другом пакете

## Краткое объяснение

### private

Доступен только внутри того же класса.

### Без спецификатора доступа

Такой доступ еще называют package-private.

Он разрешает использовать поле или метод только внутри того же пакета.

### protected

Доступен:

- внутри того же пакета;
- в подклассах из других пакетов.

### public

Доступен из любого класса.

## Что показывает пример

- внутри `AccessDemo` доступны все члены;
- в `SamePackageTest` доступны `default`, `protected` и `public`, но не `private`;
- в `ChildInOtherPackage` доступны `protected` и `public`;
- в `OtherPackageTest` из другого пакета через обычный объект доступен только `public`.

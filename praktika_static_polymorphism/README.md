# Тема: статические методы и полиморфизм

## Практика 1

Дан пример:

```java
class A {
    public static void method() {
        ...
    }
}

class B extends A {
    @Override
    public static void method() {
        ...
    }
}
```

## Почему динамический полиморфизм не работает для static-методов

Динамический полиморфизм работает только для обычных экземплярных методов, потому что выбор метода происходит **во время выполнения программы** в зависимости от того, объект какого класса реально находится в памяти.

Со статическими методами это невозможно, потому что:

- статический метод принадлежит **классу**, а не объекту;
- статический метод выбирается **на этапе компиляции**;
- для его вызова не нужен конкретный экземпляр объекта.

То есть вызов статического метода определяется не по реальному типу объекта, а по типу ссылки или имени класса, через которое идет обращение.

## Что происходит на самом деле

Статические методы не переопределяются, а **скрываются** (`method hiding`).

Это значит, что метод в подклассе не заменяет метод суперкласса во время выполнения, как это происходит у обычных методов.

## Пример

```java
class A {
    public static void method() {
        System.out.println("A");
    }
}

class B extends A {
    public static void method() {
        System.out.println("B");
    }
}

public class Main {
    public static void main(String[] args) {
        A obj = new B();
        obj.method();
    }
}
```

В этом случае будет выведено:

```text
A
```

Почему:

- переменная `obj` имеет тип `A`;
- вызов `obj.method()` для статического метода определяется как `A.method()`;
- реальный объект `new B()` здесь не влияет на выбор метода.

## Важное замечание

Аннотацию `@Override` в таком коде ставить нельзя, потому что `static`-метод не переопределяет метод суперкласса.

Программа с `@Override` над статическим методом не скомпилируется.

## Вывод

Динамический полиморфизм нельзя применить к статическим методам, потому что они:

- принадлежат классу, а не объекту;
- выбираются на этапе компиляции;
- не переопределяются, а только скрываются.

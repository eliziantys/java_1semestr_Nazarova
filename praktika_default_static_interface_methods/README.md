# Тема: default и static методы интерфейсов

В этой папке:

- `Practice1.java` — пример интерфейса с `default` и `static` методом
- `Practice2.java` — пример конфликта одинаковых `default`-методов из двух интерфейсов

## Практика 1

В `Practice1.java` интерфейс `Info` содержит:

- один неабстрактный метод `show()`, объявленный как `default`;
- один статический метод `printStatic()`.

Класс `InfoImpl` реализует этот интерфейс и переопределяет `show()`.

### Способы вызова методов интерфейса

#### Вызов `default` метода

`default` метод можно вызвать:

- через объект реализующего класса;
- через ссылку типа интерфейса.

Примеры:

```java
obj.show();
ref.show();
```

#### Вызов `static` метода

Статический метод интерфейса вызывается только через имя интерфейса:

```java
Info.printStatic();
```

Через объект класса такой метод вызывать нельзя.

## Практика 2

Если класс реализует два интерфейса, в которых есть одинаковые `default` методы, то возникает конфликт.

Пример:

- `First.show()`
- `Second.show()`

Класс `Demo` не может унаследовать оба варианта автоматически.  
Поэтому он **обязан сам переопределить** этот метод.

Пример решения:

```java
@Override
public void show() {
    ...
}
```

При необходимости внутри этого метода можно явно вызвать реализацию нужного интерфейса:

```java
First.super.show();
Second.super.show();
```

## Вывод

- `default` методы можно наследовать и переопределять;
- `static` методы интерфейса вызываются только через имя интерфейса;
- если два интерфейса содержат одинаковый `default` метод, класс обязан явно разрешить конфликт.

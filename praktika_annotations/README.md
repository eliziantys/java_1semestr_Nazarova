# Тема: аннотации

В этой папке:

- `Practice1.java` — примеры использования `@Override`, `@Deprecated`, `@SuppressWarnings`

## Практика 1

### `@Override`

Аннотация `@Override` показывает, что метод подкласса переопределяет метод суперкласса.

Она нужна для:

- повышения читаемости кода;
- проверки компилятором, что переопределение действительно выполнено правильно;
- обнаружения ошибок в сигнатуре метода.

Пример:

```java
@Override
public void show() {
    System.out.println("Метод подкласса");
}
```

Если такого метода в суперклассе нет, компилятор покажет ошибку.

### `@Deprecated`

Аннотация `@Deprecated` помечает устаревший элемент программы:

- метод;
- класс;
- поле;
- конструктор.

Она нужна, чтобы показать, что элемент лучше больше не использовать, потому что существует новый, более правильный или безопасный вариант.

Пример:

```java
@Deprecated
public void oldMethod() {
    System.out.println("Устаревший метод");
}
```

При использовании такого метода компилятор обычно выдает предупреждение.

### `@SuppressWarnings`

Аннотация `@SuppressWarnings` подавляет предупреждения компилятора.

Она нужна, когда программист осознанно использует конструкцию, которая вызывает предупреждение, и не хочет видеть это предупреждение.

Примеры:

```java
@SuppressWarnings("deprecation")
```

Подавляет предупреждение об использовании устаревшего метода.

```java
@SuppressWarnings("rawtypes")
```

Подавляет предупреждение о raw type.

## Вывод

- `@Override` помогает проверить правильность переопределения;
- `@Deprecated` помечает устаревший код;
- `@SuppressWarnings` подавляет предупреждения компилятора.

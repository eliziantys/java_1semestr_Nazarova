# Тема: внутренние классы

В этой папке:

- `Practice1.java` — пример внутренних классов с разными спецификаторами доступа

## Практика 1

Во внешнем классе можно объявлять внутренние классы со следующими вариантами доступа:

- `public`
- `protected`
- без спецификатора доступа
- `private`

В `Practice1.java` показаны:

- `PublicInner`
- `ProtectedInner`
- `DefaultInner`
- `PrivateInner`

### Как они ведут себя

- `public` внутренний класс доступен везде, где доступен внешний класс;
- `protected` внутренний класс доступен в пределах пакета и в подклассах;
- внутренний класс без спецификатора доступа доступен только в пределах пакета;
- `private` внутренний класс доступен только внутри внешнего класса.

## Практика 2

Да, внутренний класс имеет доступ ко **всем** полям и методам внешнего класса, включая:

- `private`
- доступ по умолчанию
- `protected`
- `public`

В `Practice1.java` это показано в классе `PublicInner`, где внутренний класс обращается ко всем полям и методам внешнего класса:

```java
System.out.println(secret);
privateMethod();
```

Спецификатор доступа внешних полей и методов для внутреннего класса значения не меняет: внутренний класс все равно может обращаться ко всем членам своего внешнего класса.

## Практика 3

Да, внешний класс тоже имеет доступ ко **всем** полям и методам внутреннего класса, даже если они `private`.

В `Practice1.java` это показано в методе:

```java
public void accessPrivateInner()
```

Там внешний класс создает объект `PrivateInner` и обращается к его `private` полю и `private` методу:

```java
System.out.println(inner.value);
inner.print();
```

То есть между внешним и внутренним классом действует более тесная связь, чем между обычными разными классами.

## Вывод

- доступность самого внутреннего класса снаружи зависит от его спецификатора доступа;
- внутренний класс имеет доступ ко всем членам внешнего класса;
- внешний класс имеет доступ ко всем членам внутреннего класса.

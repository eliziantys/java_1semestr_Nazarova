# Практика #1

1. Показана обработка нескольких исключений одинаковым способом
   через catch с оператором |.

2. Показана иерархия исключений:
   Ex1 <- Ex2 <- Ex3.

3. Нельзя объединять через | исключения из одной иерархии,
   например:
   catch (Ex1 | Ex2 e)
   — это вызовет ошибку компиляции.

# Практика #2

final в catch означает, что переменную e нельзя переназначить.

Пример запрещённой операции:

e = new Exception();

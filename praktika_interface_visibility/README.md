# Тема: область видимости интерфейсов

## Практика 1

Дан интерфейс:

```java
interface Interface {
  ...
}
```

Такой интерфейс имеет область видимости **по умолчанию** (`package-private`).

Это значит:

- он доступен только внутри того же пакета;
- из другого пакета использовать его нельзя.

## Практика 2

Дан интерфейс:

```java
public interface Interface {
  ...
}
```

Такой интерфейс имеет область видимости **public**.

Это значит:

- он доступен из любого пакета;
- его можно использовать в любых других классах, если он импортирован.

## Практика 3

Дан интерфейс:

```java
protected interface Interface {
  ...
}
```

Для **верхнеуровневого интерфейса** такой вариант невозможен.

Программа не скомпилируется, потому что верхнеуровневый интерфейс может иметь только:

- `public`
- доступ по умолчанию

Спецификатор `protected` можно использовать только для **вложенного интерфейса**, то есть интерфейса, который объявлен внутри класса.

## Практика 4

Дан интерфейс:

```java
private interface Interface {
  ...
}
```

Для **верхнеуровневого интерфейса** такой вариант тоже невозможен.

Программа не скомпилируется, потому что верхнеуровневый интерфейс не может быть `private`.

Спецификатор `private` можно использовать только для **вложенного интерфейса** внутри другого класса.

## Итог

Для верхнеуровневого интерфейса допустимы только два варианта:

- без спецификатора доступа — доступ только в пределах пакета;
- `public` — доступ из любого пакета.

Спецификаторы:

- `protected`
- `private`

для верхнеуровневого интерфейса недопустимы.

# Тема: перегрузка, переопределение и полиморфизм

В этой папке:

- `Practice1.java` — пример перегруженных методов
- `Practice2.java` — пример переопределенного метода и динамического связывания
- `Practice3.java` — пример, где `@Override` помогает обнаружить ошибку

## Практика 1

Перегрузка — это методы с одинаковым именем, но с разными параметрами.

В `Practice1.java` метод `sum` перегружен так:

- `sum(int a, int b)`
- `sum(int a, int b, int c)`
- `sum(double a, double b)`

## Практика 2

Переопределение — это ситуация, когда в подклассе создается метод с тем же именем и теми же параметрами, что и в суперклассе.

В `Practice2.java`:

- в `Animal` есть метод `sound()`;
- в `Cat` этот метод переопределен.

Когда выполняется код:

```java
Animal a2 = new Cat();
a2.sound();
```

во время выполнения вызывается метод класса `Cat`, потому что объект реально является экземпляром `Cat`.

Это и есть динамическое связывание.

### Что будет, если не совпадает тип возвращаемого значения

Если у метода подкласса не совпадает возвращаемый тип с методом суперкласса, то обычное переопределение не получится.

Программа выдаст ошибку компиляции, если возвращаемый тип несовместим.

Например, так нельзя:

```java
class A {
    public int getValue() {
        return 1;
    }
}

class B extends A {
    public String getValue() {
        return "1";
    }
}
```

Потому что методы отличаются только возвращаемым типом, а этого недостаточно ни для перегрузки, ни для корректного переопределения.

## Практика 3

Аннотация `@Override` помогает обнаружить ошибку, когда программист думает, что переопределяет метод, а на самом деле пишет другой метод.

В `Practice3.java` в комментарии показан такой пример:

```java
@Override
public void print(int text) {
    ...
}
```

В суперклассе метод имеет вид:

```java
public void print(String text)
```

То есть параметр отличается: `int` вместо `String`.

Без `@Override` компилятор посчитал бы это просто новым методом в подклассе.  
С `@Override` компилятор сразу покажет ошибку и сообщит, что такой метод не переопределяет метод суперкласса.

## Вывод

- перегрузка определяется на этапе компиляции;
- переопределение работает во время выполнения программы;
- `@Override` помогает не допустить ошибку в сигнатуре метода.

# Тема: комментарии и javadoc

## Практика 1

`javadoc` — это стандартная утилита Java для автоматического создания документации по исходному коду.

Она считывает специальные комментарии вида `/** ... */`, которые размещаются перед:

- классами
- методами
- полями

На основе этих комментариев `javadoc` создает HTML-документацию.

### Основные дескрипторы

- `@author` — автор
- `@version` — версия
- `@param` — параметр метода
- `@return` — возвращаемое значение
- `@deprecated` — устаревший элемент
- `@since` — с какой версии существует элемент
- `@throws` или `@exception` — возможное исключение
- `@see` — ссылка на связанный элемент

### Пример комментария javadoc

```java
/**
 * Класс для приветствия пользователя.
 * @author Student
 * @version 1.0
 */
public class Hello {

    /**
     * Метод выводит приветствие.
     * @param name имя пользователя
     * @return строка приветствия
     */
    public String sayHello(String name) {
        return "Привет, " + name;
    }
}
```

Готовый краткий пример в коде находится в файле `Example.java`.

### Как пользоваться javadoc

Если файл называется `Hello.java`, то документация создается командой:

```bash
javadoc Hello.java
```

После выполнения команды появляется HTML-документация, которую можно открыть в браузере.

### Вывод

Я изучил утилиту `javadoc` и выяснил, что она используется для создания документации по Java-коду на основе специальных комментариев `/** ... */`.

Для сдаваемых практических работ полезно добавлять содержательные комментарии, потому что это делает код понятнее и позволяет автоматически сформировать документацию.

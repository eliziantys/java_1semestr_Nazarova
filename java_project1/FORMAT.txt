Описание формата закодированного файла HuffmanTool

Все числовые поля записываются в порядке байт big-endian, как в DataOutputStream Java.

Структура файла:

1. Magic header, 4 байта
   ASCII-строка HUF1. Нужна для проверки, что файл действительно создан этой программой.

2. Original length, 8 байт
   Длина исходного файла в байтах.

3. Encoded bit length, 8 байт
   Точная длина закодированного потока в битах. Последний байт данных может содержать
   неиспользуемые нули справа, поэтому длина в битах хранится отдельно.

4. Dictionary size, 2 байта
   Количество различных байтов в исходном файле.

5. Dictionary entries, по 9 байт на запись
   Каждая запись состоит из:
   - 1 байт: значение исходного байта от 0 до 255;
   - 8 байт: частота появления этого байта в исходном файле.

   По частотам декодер заново строит дерево Хаффмана. При равных частотах узлы
   упорядочиваются по минимальному байту в поддереве, поэтому дерево строится
   детерминированно и одинаково при кодировании и декодировании.

6. Encoded data
   Поток битов, полученный заменой каждого исходного байта его кодом Хаффмана.
   Биты упаковываются слева направо: первый записанный бит становится старшим битом
   первого байта данных. Если количество битов не кратно 8, последний байт
   дополняется нулями справа.

Особый случай:
Если исходный файл содержит только один различный символ, ему назначается код 0.
Например, файл 1111111111 кодируется десятью битами 0000000000.

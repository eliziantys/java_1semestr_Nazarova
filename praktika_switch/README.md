## Практика 2

Если в `case` отсутствует `break`, то выполнение программы не остановится на этом `case`, а продолжится дальше, в следующих ветках `case`, пока не встретится `break` или не закончится весь `switch`.

Это поведение называется "проваливание" (`fall-through`).

Для данного примера:

```java
case 2:
    i = 2;
case 3:
    i = 3;
    break;
```

Если `value == 2`, то:

1. сначала выполнится `i = 2;`
2. потом сразу выполнится код из `case 3`
3. итоговое значение `i` будет равно `3`

То есть отсутствие `break` может привести к тому, что выполнится код из нескольких веток подряд.

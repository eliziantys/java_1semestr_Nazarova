# Тема: классы-оболочки

В этой папке:

- `Practice1.java` — примеры использования метода `decode()`
- `Practice2.java` — способы создания экземпляров `Boolean`
- `Practice4.java` — код для разбора сравнения `Integer` и работы `IntegerCache`

## Практика 1

Метод `decode()` преобразует строку в число и понимает разные системы счисления:

- обычная запись: `"25"` -> `25`
- запись с ведущим нулем: `"012"` -> `10` в десятичной системе, потому что это восьмеричное число
- шестнадцатеричная запись: `"0x12"` -> `18`
- шестнадцатеричная запись через `#`: `"#1A"` -> `26`

## Практика 2

Основные способы создания объекта `Boolean`:

- `Boolean.TRUE`
- `Boolean.FALSE`
- `Boolean.valueOf(true)`
- `Boolean.valueOf(false)`
- `Boolean.valueOf("true")`
- `Boolean.valueOf("false")`
- автоупаковка: `Boolean b = true;`

Конструкторы `new Boolean(...)` использовать не следует.

## Практика 3

`NullPointerException` при автоупаковке и автораспаковке возникает тогда, когда выполняется автораспаковка `null`.

Пример:

```java
Integer value = null;
int number = value;
```

Здесь переменная `value` равна `null`, а при присваивании в `int` Java пытается извлечь из объекта значение базового типа. Так как объекта нет, возникает `NullPointerException`.

## Практика 4

Код напечатает:

```text
a1==i1 true
b1==i1 true
a1==b1 false
a1.equals(i1) -> true
b1.equals(i1) -> true
a1.equals(b1) -> true

a2==i2 true
b2==i2 true
a2==b2 true
a2.equals(i2) -> true
b2.equals(i2) -> true
a2.equals(b2) -> true
```

Объяснение:

- при сравнении `a1 == i1` и `b1 == i1` объект `Integer` автораспаковывается в `int`, поэтому сравниваются значения, а не ссылки;
- `equals(...)` тоже сравнивает значения, поэтому в этих случаях результат `true`;
- при сравнении `a1 == b1` и `a2 == b2` сравниваются ссылки на объекты.

Почему для `128` получается `false`, а для `127` получается `true`:

- значения от `-128` до `127` Java кэширует;
- поэтому для `127` переменные `a2` и `b2` ссылаются на один и тот же объект;
- значение `128` уже не входит в стандартный диапазон кэша, поэтому создаются разные объекты.

### Для чего нужен `IntegerCache`

`IntegerCache` используется для хранения часто используемых объектов `Integer` в диапазоне от `-128` до `127`.

Это нужно для:

- экономии памяти;
- уменьшения количества создаваемых объектов;
- повышения производительности.

# Тема: наследование

## Практика 1

### Как использовать `this` и `super` в подклассах

В подклассах:

- `this` используется для обращения к текущему объекту;
- `super` используется для обращения к членам суперкласса.

### Использование `this`

`this` позволяет:

- обращаться к полям и методам текущего объекта;
- отличать поле класса от параметра метода или конструктора;
- вызывать другой конструктор этого же класса через `this(...)`.

Пример:

```java
class Parent {
    protected int x = 10;
}

class Child extends Parent {
    private int x;

    public Child(int x) {
        this.x = x;
    }

    public void print() {
        System.out.println(this.x);
    }
}
```

Здесь `this.x` означает поле текущего объекта `Child`.

### Использование `super`

`super` позволяет:

- обращаться к полям суперкласса;
- вызывать методы суперкласса;
- вызывать конструктор суперкласса через `super(...)`.

Пример:

```java
class Parent {
    protected int x = 10;

    public void show() {
        System.out.println("Parent");
    }
}

class Child extends Parent {
    protected int x = 20;

    public void print() {
        System.out.println(this.x);
        System.out.println(super.x);
    }

    @Override
    public void show() {
        super.show();
        System.out.println("Child");
    }
}
```

Здесь:

- `this.x` — поле подкласса;
- `super.x` — поле суперкласса;
- `super.show()` — вызов метода суперкласса.

### Главное различие

- `this` относится к текущему объекту;
- `super` относится к родительской части текущего объекта.

## Практика 2

Даны классы:

```java
class A {
  int a1;
  public int a2;
  protected int a3;
  private int a4;

  void method1() {
    ...
  }

  public void method2() {
    ...
  }

  protected void method3() {
    ...
  }
  
  private void method4() {
    ...
  }
}

class B extends A {
  ...
}

class C extends B {
  ...
}
```

### Что происходит при наследовании

#### `private`

Члены `private` наследуются как часть объекта, но **напрямую недоступны** в подклассах `B` и `C`.

Это значит:

- `a4` нельзя использовать в `B` и `C`;
- `method4()` нельзя вызывать в `B` и `C`.

Они доступны только внутри самого класса `A`.

#### Без спецификатора доступа

Члены без спецификатора доступа доступны в пределах того же пакета.

Если классы `A`, `B`, `C` находятся в одном пакете, то:

- `a1` доступно в `B` и `C`;
- `method1()` доступен в `B` и `C`.

Если подклассы находятся в другом пакете, такой доступ пропадает.

#### `protected`

Члены `protected` доступны:

- в том же пакете;
- в подклассах.

Значит:

- `a3` доступно в `B`;
- `a3` доступно в `C`;
- `method3()` доступен и в `B`, и в `C`.

#### `public`

Члены `public` доступны везде.

Значит:

- `a2` доступно в `B` и `C`;
- `method2()` доступен в `B` и `C`.

## Итог по примеру

Если считать, что классы находятся в одном пакете, то в `B` и `C`:

- доступны `a1`, `a2`, `a3`;
- недоступно `a4`;
- доступны `method1()`, `method2()`, `method3()`;
- недоступен `method4()`.

Если классы-потомки находятся в другом пакете, то:

- доступ без спецификатора пропадает;
- `protected` остается доступным для подклассов;
- `public` остается доступным;
- `private` остается недоступным.

## Краткий вывод

- `private` — только внутри своего класса;
- без спецификатора — только внутри пакета;
- `protected` — внутри пакета и в подклассах;
- `public` — везде.

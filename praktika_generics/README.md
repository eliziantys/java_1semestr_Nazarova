# Тема: параметризация (generics)

В этой папке:

- `Practice1.java` — пример использования `instanceof` с параметризованными классами
- `Practice2.java` — примеры `<? extends T>` и `<? super T>`

## Практика 1

С объектами параметризованных классов оператор `instanceof` можно использовать только по отношению к самому generic-классу или к wildcard-варианту.

Правильные примеры:

```java
obj instanceof Wrapper<?>
strWrapper instanceof Wrapper<?>
```

Неправильные примеры:

```java
obj instanceof Wrapper<Integer>
strWrapper instanceof Wrapper<String>
```

Так делать нельзя, потому что в Java используется стирание типов (`type erasure`). После компиляции информация о конкретном параметре типа (`Integer`, `String`) не сохраняется в таком виде, поэтому проверить его через `instanceof` нельзя.

## Практика 2

### `<? extends T>`

Пример:

```java
List<? extends Number>
```

Это означает: список объектов некоторого типа, который наследуется от `Number`.

Туда могут подойти:

- `List<Integer>`
- `List<Double>`
- `List<Float>`

Такой вариант нужен, когда данные в основном **читаются**.

Из такого списка удобно получать элементы как `Number`, но нельзя безопасно добавлять туда новые объекты конкретного типа.

### `<? super T>`

Пример:

```java
List<? super Integer>
```

Это означает: список объектов некоторого типа, который является родителем для `Integer`.

Туда могут подойти:

- `List<Integer>`
- `List<Number>`
- `List<Object>`

Такой вариант нужен, когда данные в основном **записываются**.

В такой список можно безопасно добавлять `Integer`, но при чтении элементы обычно приходится рассматривать как `Object`.

## Зачем они нужны

- `<? extends T>` используют, когда нужно принимать коллекции потомков и читать из них значения;
- `<? super T>` используют, когда нужно записывать объекты в коллекцию родительского типа.

Короткое правило:

- `extends` — "можно читать";
- `super` — "можно записывать".

## Важное замечание

В обобщении класса пишут:

```java
class Wrapper<T extends Number>
```

А конструкция `super` используется не в объявлении типа параметра класса, а в wildcard-ограничениях, например:

```java
List<? super Integer>
```

# Тема: логические блоки

## Практика 1

Дан код:

```java
public class A {
    {
        System.out.println("logic (1) id= " + this.id);
    }

    static {
        System.out.println("static logic");
    }

    private int id = 1;

    public A(int id) {
        this.id = id;
        System.out.println("ctor id= " + id);
    }

    {
        System.out.println("logic (2) id= " + id);
    }
}

public class Main {
    public static void main(String[] args) {
        new A(100);
    }
}
```

### Что выведет программа

```text
static logic
logic (1) id= 0
logic (2) id= 1
ctor id= 100
```

### Почему именно так

При создании объекта все происходит в таком порядке:

1. сначала один раз выполняется `static` блок;
2. затем для нового объекта последовательно выполняются нестатические логические блоки и инициализация полей в порядке их расположения в классе;
3. после этого вызывается конструктор.

Разберем по шагам:

1. Выполняется `static` блок:

```java
static {
    System.out.println("static logic");
}
```

Поэтому сначала выводится:

```text
static logic
```

2. Потом создается объект `A`, и выполняется первый логический блок:

```java
{
    System.out.println("logic (1) id= " + this.id);
}
```

В этот момент поле `id` еще не инициализировано строкой `private int id = 1;`, поэтому оно имеет значение по умолчанию для типа `int`, то есть `0`.

Поэтому выводится:

```text
logic (1) id= 0
```

3. Затем выполняется инициализация поля:

```java
private int id = 1;
```

Теперь `id` становится равным `1`.

4. После этого выполняется второй логический блок:

```java
{
    System.out.println("logic (2) id= " + id);
}
```

Теперь выводится:

```text
logic (2) id= 1
```

5. И только потом вызывается конструктор:

```java
public A(int id) {
    this.id = id;
    System.out.println("ctor id= " + id);
}
```

Параметр конструктора равен `100`, поэтому поле `this.id` получает значение `100`, и выводится:

```text
ctor id= 100
```

## Вывод

Нестатические логические блоки и инициализация полей выполняются до конструктора и строго в порядке расположения в классе.  
Статический блок выполняется раньше и только один раз.
